2. 改善清單（優先級排序）

優先級 改善項目 問題 建議
P1 儀表板缺少營運圖表與主管摘要 首頁有 KPI，但缺乏一眼判斷趨勢的視覺總覽。 新增圖表區與今日焦點面板，提升晨會/主管巡檢效率。
P1 手機/窄螢幕體驗不足 儀表板 4 欄固定，手機/小螢幕可讀性弱。 加入 responsive breakpoint，讓 KPI、模組卡、雙欄自適應。
P1 登入錯誤回饋不夠標準化 使用者知道失敗，但測試與無障礙訊息不夠穩定。 統一錯誤提示 class 與 ARIA alert，提升 UX 與自動測試穩定性。
P2 訂單中心搜尋/篩選體驗仍可再強化 已有功能，但測試自動化互動仍不穩定。 後續改成更明確 selector、狀態列、結果計數與空狀態提示。
P2 測試腳本缺少 npm 快捷指令 每次測試要手敲 node 指令，不利交付前檢查。 補上 npm test 與 headless 模式。
P3 主控台與 HTML 版 dashboard 模組入口不一致 Notion 版入口完整，HTML 版主控台仍只有 3 個模組。 下一輪統一兩套 dashboard 的模組清單與權限邏輯。
